Level-of-Detail Preprocessor

Aim

The aim of this section of the project was to build an effective Preprocessor component capable of converting a 3D model, in *.ply format, into a hierarchical mesh file. After researching current Level-of-Detail techniques, we decided on using a Hierarchical Level-of-Detail (HLOD) scheme as an effective method of rendering very large meshes. The HLOD scheme represents an input mesh as a hierarchical tree of subdivided meshes in which each node represents a simplified version of all of its combined child nodes. The HLOD scheme allows detail to be gained in any region of the model by simply traversing the tree in order to reach a node which contains the requried level of detail mesh for the region.

The core research goal of this section was to determine which hierarchical structures were most suitable for generating HLOD structures. We considered unbalanced structures, such as Octrees and KD-Trees, as well as balanced structures such as Variable KD-Trees and Multiway KD-Trees.
