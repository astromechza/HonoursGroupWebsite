Overview

[Insert Zamani blurb here.]

Our solution is based on a hierarchical level-of-detail scheme which improves rendering performance by only enhancing detail on parts of the model that are close to the viewer. This saves system resources by loading less data and rendering polygons per frame means the less strain on the GPU.

Our project was broken down into the following components:

  • The preprocessor which converts the original models into the hierarchical file format (by Benjamin Meier).
  • The simplifier which reduces the number of polygons required to represent a given mesh (by Daniel Burnham-king).
  • The renderer which renders the hierarchy file output by the preprocessor and calculates the required level-of-detail (by Justin Cossutti).
